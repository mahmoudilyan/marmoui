<Accordions>
  <Accordion title="Overview">
    Select is a basic single-selection dropdown built on Radix UI primitives. It renders natively and does not include search or multi-select.
  </Accordion>
</Accordions>

## When to use

<GuidelinesGrid>
	<Guideline type="do" title="Use for short single-choice lists (3–10 options)">
		Status, priority, role, billing cycle, currency. Options the user already knows the meaning of.
	</Guideline>
	<Guideline type="do" title="Use for form fields where exactly one value is needed">
		Required dropdowns inside a form. Pair with `Field` for label + error.
	</Guideline>
	<Guideline type="dont" title="Don't use for long lists">
		10+ options become unscannable in a Select. Use **SelectSearchable** so the user can type to filter.
	</Guideline>
	<Guideline type="dont" title="Don't use for multi-select">
		Multiple selectable options → **SelectSearchable** with `multiple`. Or **CheckboxGroup** if the list is short and always visible.
	</Guideline>
	<Guideline type="dont" title="Don't use for action menus">
		"More actions" / "Open in new tab" / etc. → **Dropdown** (which is a menu, not a value picker).
	</Guideline>
</GuidelinesGrid>

## Choosing the right alternative

| You want… | Use this |
|---|---|
| Long searchable single-select | `SelectSearchable` |
| Searchable multi-select | `SelectSearchable` with `multiple` |
| Searchable checkbox list | `SelectCheckboxSearchable` |
| Multi-category pickers (e.g. tags + segments) | `SelectMultiCategory` |
| 2–4 mutually exclusive options always visible | `RadioGroup` (or `SegmentedControl` for compact) |
| 2 mutually exclusive options always visible | `Switch` if it's an on/off — otherwise `RadioGroup` |
| "More actions" menu (Edit / Delete / Archive…) | `Dropdown` |
| Combobox / autocomplete from free text | `SelectSearchable` with `creatable` |

## Choosing the Right Select

| Component | Search | Multi-select | Indicator | Best for |
|-----------|--------|-------------|-----------|----------|
| **Select** | No | No | Checkmark (right) | Short lists (3–10 items), form fields |
| **SelectSearchable** | Yes | Optional | Checkmark (right) | Long lists, tagging, team assignment |
| **SelectSearchable** `indicator="checkbox"` | Yes | Optional | Checkbox (left) | Column toggles, permissions, settings |
| **SelectMultiCategory** | Yes | Yes | Checkbox | Categorized multi-select (recipients, segments) |

**Indicator guidance:**
- Use **checkmark on the right** (`indicator="check"`, default) for standard selection — the user picks a value and the check confirms it.
- Use **checkbox on the left** (`indicator="checkbox"`) when items represent toggleable states — the user mentally models each item as enabled/disabled, like toggling table columns or permissions.

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Use for short, known lists">
		Select works best for 3–10 options the user already knows (status, priority, role). For longer lists, use SelectSearchable.
	</Guideline>
	<Guideline type="do" title="Use groups for related options">
		Group related options with SelectGroup and SelectLabel when you have distinct categories.
	</Guideline>
	<Guideline type="do" title="Provide clear placeholders">
		Use descriptive placeholder text that indicates what type of selection is expected.
	</Guideline>
	<Guideline type="dont" title="Don't use for binary choices">
		Use a Switch or Checkbox instead of a select with only two options.
	</Guideline>
	<Guideline type="dont" title="Don't use for long lists">
		If users need to search or scroll through many items, use SelectSearchable instead.
	</Guideline>
	<Guideline type="dont" title="Don't use for actions">
		This is for selecting values, not triggering actions. Use DropdownMenu for action menus.
	</Guideline>
</GuidelinesGrid>

## Props

### Select (Root)

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | Controlled selected value |
| `defaultValue` | `string` | - | Uncontrolled default value |
| `onValueChange` | `(value: string) => void` | - | Callback when selection changes |
| `disabled` | `boolean` | `false` | Disable the entire select |
| `name` | `string` | - | Form field name for native submission |

### SelectTrigger

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `'sm' \| 'md'` | `'md'` | Trigger height — `sm` for toolbars, `md` for forms |
| `className` | `string` | - | Additional className |

### SelectItem

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | Item value (required) |
| `disabled` | `boolean` | `false` | Disable this option |

## Accessibility

- Built on Radix UI Select primitives with full ARIA support.
- Keyboard navigation: arrow keys to navigate, Enter to select, Escape to close.
- Type-ahead: typing characters jumps to matching items.
- Focus is managed automatically when the dropdown opens and closes.
- Screen readers announce the selected value and available options.
