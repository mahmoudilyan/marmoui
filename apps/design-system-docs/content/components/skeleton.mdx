import { Skeleton } from '@marmoui/ui';

<Accordions>
  <Accordion title="Overview">
    Skeleton displays animated placeholder shapes that match the layout of the content being loaded. This approach gives users a sense of the page structure before data arrives.

    **When to Use:**
    - Loading states for cards, lists, and page sections
    - Initial data fetch before content renders
    - When the content layout is predictable

    **When NOT to Use:**
    - For unknown content shapes (use Spinner)
    - For very fast loads (under 300ms)
    - For error states (use EmptyState or Alert)
  </Accordion>
</Accordions>

<Installation />

## Usage

```tsx
import { Skeleton } from '@marmoui/ui';
```

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Match content layout">
		Skeletons should closely match the shape and size of the final content.
	</Guideline>
	<Guideline type="do" title="Use for predictable layouts">
		Skeleton works best when you know the structure of the incoming content.
	</Guideline>
	<Guideline type="dont" title="Don't show for fast loads">
		If content loads in under 300ms, skip the skeleton to avoid flickering.
	</Guideline>
	<Guideline type="dont" title="Don't animate too aggressively">
		The pulse animation should be subtle and calming, not distracting.
	</Guideline>
</GuidelinesGrid>

## Accessibility

- Skeleton elements have `aria-hidden="true"` as they are purely decorative.
- Provide an `aria-busy="true"` on the container element while loading.
- Announce loading state to screen readers with `aria-live` regions.
