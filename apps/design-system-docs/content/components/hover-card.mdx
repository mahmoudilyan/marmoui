import { HoverCard, HoverCardTrigger, HoverCardContent, Avatar, AvatarFallback } from '@marmoui/ui';

<Accordions>
  <Accordion title="Overview">
    HoverCard shows a floating card with preview content when the user hovers over a trigger element. It is useful for providing additional context about links, users, or entities.

    **When to Use:**
    - User profile previews on hover over names or avatars
    - Link previews showing destination content
    - Entity summaries in lists and tables

    **When NOT to Use:**
    - For interactive content (use Popover)
    - For short text hints (use Tooltip)
    - On mobile or touch devices (hover isn't available)
  </Accordion>
</Accordions>

<Installation />

## Usage

```tsx
import { HoverCard, HoverCardTrigger, HoverCardContent } from '@marmoui/ui';
```

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Keep previews concise">
		Show only the most relevant summary information in the hover card.
	</Guideline>
	<Guideline type="do" title="Provide open delays">
		Use a delay before showing the card to prevent accidental triggers.
	</Guideline>
	<Guideline type="dont" title="Don't put forms in hover cards">
		Hover cards should be informational. Use Popover for interactive content.
	</Guideline>
	<Guideline type="dont" title="Don't rely on hover cards for essential info">
		Not all devices support hover. Ensure content is accessible via other means.
	</Guideline>
</GuidelinesGrid>

## Accessibility

- Content is accessible via keyboard focus on the trigger.
- Hover card remains visible while hovering over the card itself.
- Escape key dismisses the hover card.
- Content is announced to screen readers when shown.
