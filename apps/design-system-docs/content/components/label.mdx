import { Label, Input } from '@marmoui/ui';

<Accordions>
  <Accordion title="Overview">
    Label provides accessible text identification for form inputs. Built on Radix UI's Label primitive, it automatically handles click-to-focus behavior and screen reader associations.

    **When to Use:**
    - Labeling form inputs, selects, textareas, and other form controls
    - When you need a standalone label outside of the Field or Form components

    **When NOT to Use:**
    - When using Field or FormLabel (they include labels internally)
    - For non-form descriptive text (use Text component)
  </Accordion>
</Accordions>

<Installation />

## Usage

```tsx
import { Label } from '@marmoui/ui';
```

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Always pair labels with inputs">
		Every form input should have an associated label using the `htmlFor` attribute.
	</Guideline>
	<Guideline type="do" title="Keep labels concise">
		Use clear, short labels that describe the expected input.
	</Guideline>
	<Guideline type="dont" title="Don't hide labels">
		Visible labels are essential for usability. Avoid visually hiding labels in favor of placeholders.
	</Guideline>
	<Guideline type="dont" title="Don't use labels for instructions">
		Keep labels brief. Use helper text for additional guidance.
	</Guideline>
</GuidelinesGrid>

## Accessibility

- Clicking the label focuses the associated input.
- Screen readers announce the label when the input receives focus.
- Uses the native `<label>` element for maximum compatibility.
