import { Switch, Label } from '@marmoui/ui';

<Accordions>
  <Accordion title="Overview">
    Switch provides a toggle control for binary states. Unlike checkboxes, switches take effect immediately and are best for settings that toggle between enabled and disabled.

    **When to Use:**
    - Toggling feature settings on/off
    - Enabling or disabling preferences
    - Any binary state that should take effect immediately

    **When NOT to Use:**
    - For form selections that require a submit action (use Checkbox)
    - For choosing between non-binary options (use RadioGroup or Select)
    - In long forms with many toggleable fields
  </Accordion>
</Accordions>

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Pair with a label">
		Always include a visible label that describes what the switch controls.
	</Guideline>
	<Guideline type="do" title="Use for immediate effects">
		Switches should apply changes immediately without requiring a save action.
	</Guideline>
	<Guideline type="dont" title="Don't use in forms">
		For form fields that get submitted, use Checkbox instead of Switch.
	</Guideline>
	<Guideline type="dont" title="Don't use negative labels">
		Avoid labels like "Disable notifications." Use positive framing: "Enable notifications."
	</Guideline>
</GuidelinesGrid>

## Accessibility

- Toggled with Space key when focused.
- Uses `role="switch"` with `aria-checked` for state.
- Label association via `id` and `htmlFor`.
- Focus indicator is clearly visible.
