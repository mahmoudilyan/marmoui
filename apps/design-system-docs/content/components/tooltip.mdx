import { Tooltip, Button, IconButton } from '@marmoui/ui';
import { MdInfo, MdSettings, MdHelp } from 'react-icons/md';

<Accordions>
  <Accordion title="Overview">
    Tooltip displays a short informational text when users hover over or focus on an element. It provides contextual help without cluttering the interface.

    **When to Use:**
    - Explaining icon-only buttons
    - Providing additional context for truncated text
    - Showing keyboard shortcuts
    - Supplementary information that doesn't need to persist

    **When NOT to Use:**
    - For interactive content (use Popover)
    - For critical information users must see (use inline text)
    - For long content or rich formatting
    - On touch-only devices where hover isn't available
  </Accordion>
</Accordions>

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Keep text short">
		Tooltip content should be 1-2 short sentences maximum.
	</Guideline>
	<Guideline type="do" title="Use for icon buttons">
		Every icon-only button should have a tooltip explaining its action.
	</Guideline>
	<Guideline type="dont" title="Don't put essential info in tooltips">
		If users must see the information, display it inline.
	</Guideline>
	<Guideline type="dont" title="Don't add tooltips to obvious elements">
		A button labeled "Delete" doesn't need a tooltip saying "Delete this item."
	</Guideline>
</GuidelinesGrid>

## Accessibility

- Appears on focus for keyboard users, not just hover.
- Uses `role="tooltip"` and `aria-describedby` linking.
- Can be disabled for screen readers when content is redundant.
- Respects `prefers-reduced-motion` for animations.
