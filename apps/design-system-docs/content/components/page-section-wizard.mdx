<Accordions>
  <Accordion title="Overview">
    PageSectionWizard is a page header variant designed for multi-step form flows. It replaces the standard breadcrumb-based `PageSection` with an editable title on the left and step indicators on the right.

    **Key Features:**
    - Inline-editable title (click to edit, Enter to save, Escape to cancel)
    - Step indicators showing progress through the wizard
    - Integrates with `Stepper` context for automatic step management
    - Sidebar toggle button when sidebar is collapsed
    - Works inside `SidebarProvider` for full layout integration
    - Pairs with a full-height page body where the wizard footer stays at the bottom

    **When to Use:**
    - Multi-step campaign creation flows
    - Multi-step form builders
    - Sequential setup wizards
    - Any flow where users need to see their progress

    **When NOT to Use:**
    - Standard content pages (use PageSection)
    - Single-step forms (use PageSection with actions)
    - Non-sequential processes (use Tabs or SegmentedControl)
  </Accordion>
</Accordions>

### With Stepper Context

When used inside a `Stepper` component, `PageSectionWizard` automatically reads the current step and navigation from context. No need to pass `currentStep`, `totalSteps`, or `onStepClick`.

<CodeExample
	title="Stepper Integration"
	description="Automatic step management via Stepper context"
	code={`import { Stepper, StepperContent, PageSectionWizard } from '@marmoui/ui';

function CampaignWizard() {
  const [activeStep, setActiveStep] = useState('details');

  return (
    <Stepper value={activeStep} onValueChange={setActiveStep}>
      <PageSectionWizard
        title="Campaign Builder"
        onTitleChange={(title) => updateCampaignName(title)}
        steps={[
          { id: 'details', label: 'Campaign Details' },
          { id: 'content', label: 'Email Content' },
          { id: 'audience', label: 'Select Audience' },
          { id: 'review', label: 'Review & Send' },
        ]}
      />
      <StepperContent value="details">
        <CampaignDetailsForm />
      </StepperContent>
    </Stepper>
  );
}`}
	preview={<WizardStepNavigationPreview />}
/>

### Full-Height Wizard Layout

Wizard content should fill the available page height. Keep the step content in a
scrollable flex child and place the footer after it with `mt-auto` (or sticky
bottom styling for long forms). Do not wrap the entire wizard step in `Card` by
default.

<CodeExample
	title="Wizard content with bottom footer"
	description="A wizard page structure that keeps navigation actions pinned to the bottom."

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Use descriptive step labels">
		Step labels should clearly describe what happens in each step (e.g., "Campaign Details" not "Step 1").
	</Guideline>

	<Guideline type="do" title="Enable editable titles">
		When the wizard creates a named entity (campaign, template), let users name it inline rather than in a separate field.
	</Guideline>

	<Guideline type="do" title="Keep steps to 3-5">
		More than 5 steps feels overwhelming. Group related steps or use sub-steps.
	</Guideline>

	<Guideline type="do" title="Keep the footer at the bottom">
		Use a flex column page body. The wizard content scrolls, and the footer sits after the content with `mt-auto` or sticky bottom styling.
	</Guideline>

	<Guideline type="do" title="Enable showSidebarToggle">
		Always set `showSidebarToggle` when using inside a `SidebarProvider` so users can re-open a collapsed sidebar.
	</Guideline>

	<Guideline type="dont" title="Don't use for non-sequential flows">
		If users can complete sections in any order, use Tabs instead.
	</Guideline>

	<Guideline type="dont" title="Don't mix with PageSection">
		Use one header type per page. Don't combine PageSection and PageSectionWizard.
	</Guideline>

	<Guideline type="dont" title="Don't wrap steps in Card by default">
		Wizard steps are page content. Use `Card` only for bounded widgets or summaries inside a step.
	</Guideline>
</GuidelinesGrid>

## Props

### PageSectionWizard

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | — | Wizard title text |
| `onTitleChange` | `(newTitle: string) => void` | — | Makes title editable; called on each keystroke |
| `onTitleChangeEnd` | `(newTitle: string) => void` | — | Called when editing ends (blur or Enter) |
| `titlePlaceholder` | `string` | `'Untitled'` | Placeholder text when title is empty |
| `currentStep` | `number` | — | Current step index (0-based). Auto-detected from Stepper context if omitted. |
| `totalSteps` | `number` | — | Total steps. Auto-detected from Stepper context if omitted. |
| `steps` | <TypeInfo>WizardStepItem[]</TypeInfo> | — | Step definitions for indicators. Uses Stepper context if omitted. |
| `onStepClick` | `(stepIndex: number) => void` | — | Called when a step indicator is clicked |
| `showSidebarToggle` | `boolean` | `true` | Show sidebar toggle when sidebar is collapsed |
| `stepsContent` | <TypeInfo>ReactNode</TypeInfo> | — | Custom content replacing the default step indicators |
| `className` | `string` | — | Additional className |

## Accessibility

- Title input supports keyboard editing (Enter to save, Escape to cancel)
- Step indicators are keyboard navigable when `onStepClick` is provided
- Current step is communicated to screen readers
- Sidebar toggle button has proper `aria-label`
- Editable title has an edit button with tooltip for discoverability
