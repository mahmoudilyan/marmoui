<Accordions>
  <Accordion title="Overview">
    Field is a form layout wrapper that consistently handles label placement, helper text, error messages, and optional indicators for any form input. It creates a standardized structure across all form elements.

    **When to Use:**
    - Wrapping any form input that needs a label
    - Displaying validation error messages
    - Showing helper text or character counts
    - Indicating required or optional fields

    **When NOT to Use:**
    - For inputs that don't need labels (search bars with placeholders)
    - When using react-hook-form integration (use Form components instead)
    - For non-form display elements
  </Accordion>
</Accordions>

<Installation />

## Usage

```tsx
import { Field } from '@marmoui/ui';
```

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Always provide labels">
		Every form field should have a visible label. Labels are essential for usability and accessibility.
	</Guideline>
	<Guideline type="do" title="Write helpful error messages">
		Error text should explain what went wrong and how to fix it.
	</Guideline>
	<Guideline type="dont" title="Don't use placeholder as label">
		Placeholders disappear when typing. Always use a visible label above the input.
	</Guideline>
	<Guideline type="dont" title="Don't show errors prematurely">
		Only display error messages after the user has interacted with the field.
	</Guideline>
</GuidelinesGrid>
