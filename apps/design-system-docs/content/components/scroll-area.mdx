import { ScrollArea, ScrollBar } from '@marmoui/ui';

<Accordions>
  <Accordion title="Overview">
    ScrollArea provides a scrollable container with custom-styled scrollbars that look consistent across all browsers. It supports both vertical and horizontal scrolling.

    **When to Use:**
    - Long lists or content that needs scrollable containment
    - Dropdown content areas
    - Sidebars with overflow content
    - Any constrained area with potentially overflowing content

    **When NOT to Use:**
    - For full-page scroll (use native scroll)
    - When content always fits (no need for scroll)
    - For horizontal carousels (use dedicated carousel patterns)
  </Accordion>
</Accordions>

<Installation />

## Usage

```tsx
import { ScrollArea, ScrollBar } from '@marmoui/ui';
```

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Set a clear height constraint">
		Always define a max-height or height for the scroll area so scrolling activates.
	</Guideline>
	<Guideline type="do" title="Indicate scrollable content">
		Use visual cues (shadows, fade) to hint that content extends beyond the visible area.
	</Guideline>
	<Guideline type="dont" title="Don't nest scroll areas">
		Multiple nested scrollable areas create confusing interaction patterns.
	</Guideline>
	<Guideline type="dont" title="Don't use for small content">
		If content always fits, don't wrap it in a scroll area.
	</Guideline>
</GuidelinesGrid>

## Accessibility

- Scrollbar is keyboard accessible with Arrow keys when focused.
- Works with native browser scroll behavior for assistive technologies.
- Custom scrollbar is visible for mouse users without affecting keyboard behavior.
