<Accordions>
  <Accordion title="Overview">
    A comprehensive collection of input components for forms, designed to handle various input scenarios from simple text entry to complex grouped inputs with actions.

    **Components Included:**
    - **Input**: Basic text input with variants and sizes
    - **InputGroup**: Compound input with addons, buttons, and icons
    - **InputShortcode**: Input with shortcode picker and emoji support
    - **NumberInput**: Numeric input with increment/decrement controls
    - **TagsInput**: Multi-value input for tags/chips
  </Accordion>
</Accordions>

## When to use

<GuidelinesGrid>
	<Guideline type="do" title="Use for free-text single-line entry">
		Names, emails, URLs, search queries, short answers. Anything that fits on one visible line.
	</Guideline>
	<Guideline type="do" title="Wrap in a Field component">
		`Field` adds the label, helper text, and error display. Always pair `Input` with `Field` in forms — never render an Input bare with a separate `<label>`.
	</Guideline>
	<Guideline type="do" title="Use the right specialised variant when relevant">
		`InputGroup` for addons (currency, units), `InputShortcode` for templating fields, `NumberInput` for numeric, `TagsInput` for multi-value.
	</Guideline>
	<Guideline type="dont" title="Don't use for multi-line content">
		Anything more than one line → **Textarea**.
	</Guideline>
	<Guideline type="dont" title="Don't use for predefined option choice">
		If the user picks from a known list → **Select** (short) or **SelectSearchable** (long). Never an Input with a datalist.
	</Guideline>
	<Guideline type="dont" title="Don't roll your own number inputs">
		Use `NumberInput` (from `@marmoui/ui`) — it handles min/max, increment/decrement, and locale-aware formatting.
	</Guideline>
</GuidelinesGrid>

## Choosing the right alternative

| You want… | Use this |
|---|---|
| Multi-line free-text (descriptions, notes, body) | `Textarea` |
| Pick one option from a short known list | `Select` |
| Pick one option from a long searchable list | `SelectSearchable` |
| Pick multiple options | `SelectSearchable` with `multiple` |
| Multi-value tag/chip entry | `TagsInput` |
| Numeric input with stepper | `NumberInput` |
| Input with currency/unit prefix | `InputGroup` |
| Input with template variable picker | `InputShortcode` |
| On/off toggle | `Switch` |
| Single yes/no opt-in | `Checkbox` |
| Search bar | `Input` with `leftIcon={<MdSearch />}` |
| Date / time | `DatePicker` |

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Use appropriate variants">
		Use destructive variant for errors, success for valid states, and warning for cautions.
	</Guideline>
	
	<Guideline type="do" title="Provide clear placeholders">
		Use descriptive placeholder text that indicates the expected input format.
	</Guideline>
	
	<Guideline type="do" title="Use icons meaningfully">
		Icons in adornments should reinforce the input's purpose (search icon for search, lock for password).
	</Guideline>
	
	<Guideline type="do" title="Match sizes consistently">
		Use the same size for related form elements (inputs, buttons, selects).
	</Guideline>
	
	<Guideline type="dont" title="Don't use too many adornments">
		Limit adornments to avoid cluttering the input. Usually one start and one end is sufficient.
	</Guideline>
	
	<Guideline type="dont" title="Don't hide essential information">
		Don't use placeholder text for important labels or instructions.
	</Guideline>
</GuidelinesGrid>

## Props

### Input

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'normal' \| 'destructive' \| 'success' \| 'warning'` | `'normal'` | Visual variant |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Input size |
| `startAdornment` | `ReactNode` | - | Content before input |
| `endAdornment` | `ReactNode` | - | Content after input |
| `className` | `string` | - | Additional className |

### InputGroup

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Size of the group |
| `className` | `string` | - | Additional className |

### InputGroupAddon

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `align` | `'inline-start' \| 'inline-end' \| 'block-start' \| 'block-end'` | `'inline-start'` | Position of addon |

### InputShortcode

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | Input value |
| `onChange` | `(value: string) => void` | - | Change handler |
| `shortcodes` | `ShortcodeGroup[]` | - | Shortcode definitions |
| `enableEmoji` | `boolean` | `false` | Enable emoji picker |
| `onLoadShortcodes` | `() => Promise<ShortcodeGroup[]>` | - | Async shortcode loader |

### NumberInput

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `number` | - | Current value |
| `onValueChange` | `(value: number) => void` | - | Change handler |
| `min` | `number` | - | Minimum value |
| `max` | `number` | - | Maximum value |
| `step` | `number` | `1` | Increment step |
| `precision` | `number` | `0` | Decimal precision |
| `variant` | `'normal' \| 'destructive' \| 'success'` | `'normal'` | Visual variant |

## Accessibility

<InfoCallout type="info" title="Form Labels">
Always pair inputs with Label components for accessibility. Use `htmlFor` to associate labels with inputs.
</InfoCallout>

- All inputs support standard ARIA attributes
- InputGroup maintains focus styling for the group
- TagsInput supports keyboard navigation and deletion
- NumberInput supports keyboard increment/decrement with arrow keys
- Adornment buttons should have `aria-label` for screen readers
