import { Text } from '@marmoui/ui';

<Accordions>
  <Accordion title="Overview">
    Text is a polymorphic typography component that maps the design system's type scale to semantic HTML elements. It supports headings, body text, and caps styles with weight, alignment, and truncation controls.

    **When to Use:**
    - Rendering any text content with design system typography
    - Headings, body text, captions, and labels
    - When you need consistent text styling with semantic HTML

    **When NOT to Use:**
    - For interactive text (use Button or Link)
    - For form labels (use Label)
    - For raw HTML when custom styling is required
  </Accordion>
</Accordions>

<Installation />

## Usage

```tsx
import { Text } from '@marmoui/ui';
```

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Use semantic elements">
		Use the `as` prop to render appropriate HTML elements (h1-h6, p, span).
	</Guideline>
	<Guideline type="do" title="Follow the type hierarchy">
		Use the heading scale in order. Don't skip levels (h1 to h3) for accessibility.
	</Guideline>
	<Guideline type="dont" title="Don't style text with raw classes">
		Use the Text component variants instead of applying typography classes directly.
	</Guideline>
	<Guideline type="dont" title="Don't use headings for visual size only">
		Choose heading levels based on document structure, not visual appearance.
	</Guideline>
</GuidelinesGrid>

## Accessibility

- Use semantic HTML elements via the `as` prop for proper document structure.
- Maintain heading hierarchy (h1 > h2 > h3) for screen reader navigation.
- Ensure sufficient color contrast for all text color variants.
- Use `visuallyHidden` for screen-reader-only text.
