## Overview

A button component designed specifically for displaying icons. Perfect for toolbars, action menus, and compact interfaces where space is at a premium.

## When to use

<GuidelinesGrid>
	<Guideline type="do" title="Use when space is limited and the icon is unambiguous">
		Toolbars, table row actions, header bars, compact data interfaces. The icon must clearly represent the action — universal icons only (close, delete, edit, share, search).
	</Guideline>
	<Guideline type="do" title="Always provide aria-label">
		Without a visible label, screen readers need `aria-label` to announce what the button does. The TypeScript type requires it.
	</Guideline>
	<Guideline type="do" title="Pair icon + text when the action isn't universally recognised">
		If users could plausibly misread the icon, use a regular **Button** with `leftIcon` instead.
	</Guideline>
	<Guideline type="dont" title="Don't use for primary CTAs">
		The main "Save", "Submit", "Continue" should be a labelled **Button**. Reserve IconButton for secondary or repeat actions.
	</Guideline>
	<Guideline type="dont" title="Don't pass an icon as children">
		Use the `icon` prop, not `<IconButton><MdEdit /></IconButton>`. The component infers sizing and a11y from the prop.
	</Guideline>
</GuidelinesGrid>

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Always provide aria-label">
		Every icon button must have an aria-label or aria-labelledby attribute to describe its purpose for screen readers.
	</Guideline>
	
	<Guideline type="do" title="Use recognizable icons">
		Choose icons that clearly communicate their action. When in doubt, use text buttons instead.
	</Guideline>
	
	<Guideline type="do" title="Size appropriately">
		Use smaller sizes (xs, sm) for toolbars and compact interfaces. Use larger sizes (md, lg) for primary actions.
	</Guideline>
	
	<Guideline type="dont" title="Don't use ambiguous icons">
		Avoid icons that users might not understand. If an icon isn't universally recognized, use a text button.
	</Guideline>
	
	<Guideline type="dont" title="Don't omit aria-label">
		Never leave an icon button without an accessible label. Screen reader users need to know what the button does.
	</Guideline>
	
	<Guideline type="dont" title="Don't use for primary actions">
		Primary page actions should use text buttons for clarity. Reserve icon buttons for secondary or tertiary actions.
	</Guideline>
</GuidelinesGrid>

## Patterns

```tsx
// Toolbar row of actions
<div className="flex items-center gap-1">
  <IconButton variant="ghost" size="sm" icon={<MdEdit />} aria-label="Edit" />
  <IconButton variant="ghost" size="sm" icon={<MdShare />} aria-label="Share" />
  <IconButton variant="ghost" size="sm" icon={<MdDelete />} aria-label="Delete" />
</div>

// With tooltip for clarity
<Tooltip content="Delete permanently">
  <IconButton variant="destructive" size="sm" icon={<MdDelete />} aria-label="Delete permanently" />
</Tooltip>

// Loading state
<IconButton icon={<MdRefresh />} aria-label="Refresh" loading={isLoading} />

// Destructive icon action
<IconButton variant="ghost-destructive" size="xs" icon={<MdClose />} aria-label="Remove tag" />
```

## Choosing the right alternative

| You want… | Use this |
|---|---|
| Labelled action with optional icon | `Button` (with `leftIcon` / `rightIcon` prop) |
| Icon + text in a tight space | `Button size="xs"` with `leftIcon` |
| Toggle state (active/inactive) | `Switch` (better signal than icon) |
| Kebab / "more actions" menu | `IconButton` with `<MdMoreVert />` inside a `Dropdown` |
| Icon-only link to another page | `IconButton asChild` wrapping a `<Link>` |
| Cluster of related icon actions | `ButtonGroup` containing `IconButton`s |

## Common Mistakes

```tsx
// ❌ WRONG — IconButton does NOT accept children as icon
<IconButton aria-label="Add"><MdAdd /></IconButton>
// ✅ CORRECT — pass icon via the `icon` prop
<IconButton icon={<MdAdd />} aria-label="Add" />

// ❌ WRONG — missing aria-label (required for accessibility)
<IconButton icon={<MdEdit />} />
// ✅ CORRECT
<IconButton icon={<MdEdit />} aria-label="Edit item" />

// ❌ WRONG — using Button with icon only (no text)
<Button><MdDelete /></Button>
// ✅ CORRECT — use IconButton for icon-only triggers
<IconButton icon={<MdDelete />} aria-label="Delete" />

// ❌ WRONG — no "rounded" or "circle" prop
<IconButton icon={<MdAdd />} rounded aria-label="Add" />
// ✅ CORRECT — apply via className if needed
<IconButton icon={<MdAdd />} aria-label="Add" className="rounded-full" />
```

## Accessibility

<InfoCallout type="warning" title="Required: ARIA Label">
Icon buttons MUST include an aria-label or aria-labelledby attribute. This is required for screen reader accessibility.
</InfoCallout>

- **Always** provide an aria-label describing the button's action
- Use clear, action-oriented labels ("Add item" not "Plus")
- Ensure sufficient touch target size (minimum 44x44px on mobile)
- Provide keyboard focus indicators
- Consider adding a tooltip for additional context
- Use appropriate color contrast for the icon
- Ensure the icon is visible in both light and dark modes
