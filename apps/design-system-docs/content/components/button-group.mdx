<Accordions>
  <Accordion title="Overview">
    ButtonGroup provides a way to group related buttons together, either visually connected (attached) or with consistent spacing (detached).

    **When to Use:**
    - Toolbar controls (text formatting, alignment)
    - View switchers (list, grid, calendar)
    - Toggle button sets
    - Segmented controls for mutually exclusive options
    - Form action groups (Save, Cancel)

    The component supports both horizontal and vertical orientations. The `size` prop is propagated to all child `Button` and `IconButton` components via context, ensuring consistent sizing with full support for padding, border radius, icon spacing, and text size.
  </Accordion>
</Accordions>

<Installation />

## Usage

Import the ButtonGroup component from the package:

```tsx
import { ButtonGroup, Button, IconButton } from '@marmoui/ui';
```

### Overriding Size on Individual Children

A child can override the group's `size` by setting its own `size` prop directly. This is useful when one button in the group needs to differ — though use this sparingly since mixed sizes break visual rhythm.

	title="Size Override"

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Use attached for related actions">
		Use attached mode when buttons represent related options in a single control (e.g., text alignment).
	</Guideline>
	
	<Guideline type="do" title="Use detached for distinct actions">
		Use detached mode for action pairs like Save/Cancel where actions are distinct but related.
	</Guideline>
	
	<Guideline type="do" title="Provide clear active states">
		When using as toggle buttons, clearly indicate the active state using the primary variant.
	</Guideline>
	
	<Guideline type="do" title="Add aria-labels to icon buttons">
		Always provide accessible labels for icon-only buttons in the group.
	</Guideline>
	
	<Guideline type="dont" title="Don't mix attached and detached">
		Use a consistent style within your application for similar use cases.
	</Guideline>
	
	<Guideline type="dont" title="Don't group unrelated actions">
		Only group buttons that are contextually related or form a cohesive control.
	</Guideline>
</GuidelinesGrid>

## Props

### ButtonGroup

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `attached` | `boolean` | `true` | Whether buttons are visually connected |
| `orientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | Layout direction |
| `size` | `'xs' \| 'sm' \| 'md' \| 'lg'` | `'md'` | Size propagated to all child buttons via context. Individual children can override with their own `size` prop. |
| `children` | `ReactNode` | - | Button or IconButton components |
| `className` | `string` | - | Additional className |

## Accessibility

<InfoCallout type="info" title="ARIA Roles">
ButtonGroup uses `role="group"` to indicate that the buttons are related. For toggle groups, consider using `role="toolbar"` or implementing a proper ARIA pattern.
</InfoCallout>

- Uses semantic grouping with `role="group"`
- Supports keyboard navigation between buttons
- Individual buttons maintain their accessibility features
- Always provide `aria-label` for icon-only buttons
- For exclusive selection, consider using `aria-pressed` to indicate state
