<Accordions>
  <Accordion title="Overview">
    Slider allows users to select a value or range by dragging a thumb along a track. It is ideal for settings where users need to choose from a continuous range of values.

    **When to Use:**
    - Adjusting volume, brightness, or opacity
    - Setting price ranges or filter thresholds
    - Configuring numerical settings with a visual range

    **When NOT to Use:**
    - For precise numeric entry (use NumberInput)
    - For very small ranges (1-3 options, use RadioGroup)
    - When exact values matter more than relative position
  </Accordion>
</Accordions>

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Show the current value">
		Display the numeric value near the slider so users know the exact selection.
	</Guideline>
	<Guideline type="do" title="Define meaningful bounds">
		Set min and max to logical limits. Don't use 0-100 if the range is 50-200.
	</Guideline>
	<Guideline type="dont" title="Don't use for precise entry">
		If users need to type exact numbers, provide a NumberInput alongside or instead.
	</Guideline>
	<Guideline type="dont" title="Don't use very small step values">
		Steps too small (e.g., 0.001) make the slider hard to control.
	</Guideline>
</GuidelinesGrid>

## Accessibility

- Keyboard controlled with Arrow keys (Left/Right for horizontal, Up/Down for vertical).
- Home/End keys jump to min/max values.
- Uses `role="slider"` with `aria-valuemin`, `aria-valuemax`, `aria-valuenow`.
- Label is associated via `aria-label` or `aria-labelledby`.
