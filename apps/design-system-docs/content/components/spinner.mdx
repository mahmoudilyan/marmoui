<Accordions>
  <Accordion title="Overview">
    Spinner provides a rotating animation that indicates ongoing processing. It is used when the system is busy and the wait time or progress is indeterminate.

    **When to Use:**
    - Button loading states
    - Page or section loading when layout is unknown
    - Inline loading indicators
    - Overlay loading states

    **When NOT to Use:**
    - When content layout is known (use Skeleton)
    - For determinate progress (use Progress)
    - For very long waits (combine with messaging)
  </Accordion>
</Accordions>

<Installation />

## Usage

```tsx
import { Spinner } from '@marmoui/ui';
```

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Add context with labels">
		Tell users what is loading when the spinner is the primary content indicator.
	</Guideline>
	<Guideline type="do" title="Match size to context">
		Use xs/sm for inline or button loading, md/lg for section loading, xl for full-page.
	</Guideline>
	<Guideline type="dont" title="Don't use for known layouts">
		If you know the content shape, use Skeleton instead for a smoother experience.
	</Guideline>
	<Guideline type="dont" title="Don't block the entire page">
		Prefer localized spinners over full-page overlays when possible.
	</Guideline>
</GuidelinesGrid>

## Accessibility

- Has `role="status"` to announce loading state to screen readers.
- The `label` prop provides `aria-label` for context.
- Visually hidden text "Loading" is announced by default.
