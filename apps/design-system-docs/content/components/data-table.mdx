    - Displaying and managing lists of records (contacts, orders, products, campaigns)

## Usage

```tsx
import { DataTable } from '@marmoui/ui';
import type { ColumnDef } from '@tanstack/react-table';
```

DataTable requires two props: `data` (array of row objects) and `columns` (TanStack Table column definitions). Everything else is optional.

### Server-Side Integration with TableState

For server-side pagination, sorting, and filtering, use the controlled `tableState` and `setTableState` props. The table communicates all state changes through `setTableState`, and you send those to your API.

<CodeExample
	title="Controlled Table State with Data Fetching"
	description="Server-side pagination, search, and filters using fetch and useEffect"
	code={`import { useState, useEffect, useMemo } from 'react';
import type { ColumnDef } from '@tanstack/react-table';
import { DataTable } from '@marmoui/ui';
import type { TableState } from '@marmoui/ui';

interface Contact {
  id: string;
  name: string;
  email: string;
  status: string;
}

function ContactsPage() {
  const [data, setData] = useState<Contact[]>([]);
  const [total, setTotal] = useState(0);
  const [isLoading, setIsLoading] = useState(true);
  const [tableState, setTableState] = useState<TableState>({
    pagination: { pageIndex: 0, pageSize: 10 },
    sorting: [],
    filters: {},
    columns: [],
    search: '',
    view: 'list',
  });

  // Fetch data whenever tableState changes
  useEffect(() => {
    const fetchContacts = async () => {
      setIsLoading(true);
      try {
        const params = new URLSearchParams({
          page: String(tableState.pagination.pageIndex),
          pageSize: String(tableState.pagination.pageSize),
          search: tableState.search || '',
          ...Object.fromEntries(
            Object.entries(tableState.filters).map(([k, v]) => [k, String(v)])
          ),
        });

        const res = await fetch(\`/api/contacts?\${params}\`);
        const json = await res.json();
        setData(json.data);
        setTotal(json.total);
      } finally {
        setIsLoading(false);
      }
    };

    fetchContacts();
  }, [tableState.pagination, tableState.search, tableState.filters]);

  const handleTableStateChange = (newState: Partial<TableState>) => {
    setTableState(prev => ({ ...prev, ...newState }));
  };

  const columns: ColumnDef<Contact>[] = [
    { accessorKey: 'name', header: 'Name', enableHiding: false },
    { accessorKey: 'email', header: 'Email' },
    { accessorKey: 'status', header: 'Status' },
  ];

  return (
    <DataTable
      data={data}
      columns={columns}
      total={total}
      dataType="contact"
      isLoadingBody={isLoading}
      tableState={tableState}
      setTableState={handleTableStateChange}
      pageSize={tableState.pagination.pageSize}
      currentPage={tableState.pagination.pageIndex}
    />
  );
}`}
	preview={<DataTableBasicPreview />}
/>

## Column Definition Reference

Columns follow the TanStack Table `ColumnDef` format:

```tsx
import type { ColumnDef } from '@tanstack/react-table';

const columns: ColumnDef<YourDataType>[] = [
  {
    id: 'uniqueId',           // Unique column ID (optional if accessorKey is set)
    accessorKey: 'fieldName', // Property name on the data object
    header: 'Display Name',   // String or render function for header
    cell: ({ row }) => ...,   // Custom cell renderer (optional)
    enableSorting: true,      // Allow sorting (default: true)
    enableHiding: true,       // Allow toggling visibility (default: true)
    size: 200,                // Column width in pixels
    meta: {
      label: 'Field Label',  // Label shown in column visibility menu
      group: 'Group Name',   // Group in column visibility menu
    },
  },
];
```

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Use custom cell renderers for readability">
		Enhance data presentation with avatars, badges, formatted dates, and links. Raw data is harder to scan.
	</Guideline>

	<Guideline type="do" title="Provide contextual empty states">
		Use the `dataType` prop (e.g., `"contact"`) so empty states say "No contacts found" instead of generic text.
	</Guideline>

	<Guideline type="do" title="Show loading states during data fetches">
		Set `isLoadingBody={true}` while fetching. Skeleton rows reassure users that data is on the way.
	</Guideline>

	<Guideline type="do" title="Pin the primary column">
		Set `enableHiding: false` on the primary identifier column (e.g., email) so users always have context.
	</Guideline>

	<Guideline type="dont" title="Don't show too many columns by default">
		Start with 4-6 visible columns. Let users toggle additional columns via the column menu.
	</Guideline>

	<Guideline type="dont" title="Don't skip pagination for large datasets">
		Always paginate when dealing with more than 50 rows. Use server-side pagination for datasets over 1,000 rows.
	</Guideline>
</GuidelinesGrid>

## Patterns

### Minimal local table (no server pagination)

Pass `data` and `columns` — sorting and client-side pagination work automatically.

```tsx
import { DataTable } from '@marmoui/ui';
import type { ColumnDef } from '@tanstack/react-table';

interface User { id: string; name: string; email: string; }

const columns: ColumnDef<User>[] = [
  { accessorKey: 'name', header: 'Name' },
  { accessorKey: 'email', header: 'Email' },
];

<DataTable columns={columns} data={users} pageSize={20} />
```

### Server-side pagination with tableState

```tsx
const [tableState, setTableState] = useState<DataTableParams>({
  page: 1, pageSize: 20, search: '', sortBy: '', sortDir: 'asc',
});

<DataTable
  columns={columns}
  data={data}
  total={totalCount}
  tableState={tableState}
  setTableState={setTableState}
  isLoadingBody={isLoading}
/>
```

### Status badge in a cell

```tsx
{
  accessorKey: 'status',
  header: 'Status',
  cell: ({ row }) => {
    const variantMap = {
      active:   'success',
      pending:  'warning',
      inactive: 'normal',
      failed:   'destructive',
    } as const;
    return (
      <Badge variant={variantMap[row.original.status]}>
        {row.original.status}
      </Badge>
    );
  },
}
```

## Common Mistakes

- **`pageSize` is a real prop** — `<DataTable columns={columns} data={data} pageSize={20} />` is correct. Default is `10`. Do not omit it when you need a specific page size.
- **`pageSize` is static** — it is the initial/controlled rows-per-page value. For server-side pagination, also pass `total`, `currentPage`, `tableState`, and `setTableState` so the table knows the full dataset size and can compute page numbers.
- Using `enableSorting` or `enablePagination` props — these do not exist; sorting is always on, pagination is controlled via `tableState`/`setTableState`.
- Forgetting `total` for server-side pagination — without it the table treats the current `data` array as the full dataset.
- Passing rendered JSX as column `header` string — use a string or a function: `header: () => <span>Name</span>`
- Not importing `ColumnDef` from `@tanstack/react-table` — it must be imported separately alongside the DataTable import.
- Using `rows` or `items` instead of `data` for the data prop — the prop name is always `data`.

## Accessibility

<InfoCallout type="info" title="Keyboard Navigation">
DataTable supports full keyboard navigation. Use Tab to move between interactive elements, Enter or Space to activate buttons, and arrow keys within dropdowns.
</InfoCallout>

- Uses proper ARIA roles (`table`, `rowgroup`, `row`, `columnheader`, `cell`)
- Sortable column headers are keyboard accessible with `aria-sort`
- Row selection checkboxes are fully accessible with `aria-checked`
- Action bar announces selected count to screen readers
- Focus management for inline action buttons and dropdown menus
- Loading and empty states are announced via `aria-live` regions
